# 少数派文章

> 标题:把旧手机变成局域网云盘:HcqDrive 让文件在 WiFi 里自由流动

---

你有没有这种时刻:想从手机里拽一张照片到电脑,但微信传会压画质、Airdroid 慢、网络上传再下载又觉得多余;或者反过来,想在电脑上给手机里塞几个文件,数据线又不在手边。

**HcqDrive 想做的就是:让这种时刻消失**。安装一个安卓应用,同一 WiFi 下的任何设备(电脑、平板、第二台手机)用浏览器打开就能访问手机里的文件——没有客户端、没有账号、文件不离开你家网络。

## 它解决什么

**场景 1:零延迟取照片。** 刚拍完一组照片想用电脑修图。打开 HcqDrive,通知栏弹出配对码和地址,电脑浏览器输入,看到 `/DCIM` 目录,挑几张下下来。整个过程不超过 10 秒,完全在局域网内。

**场景 2:从电脑往手机塞文件。** 下载了一个 PDF 想在手机上接着看,或者批量上传一堆照片备份。HcqDrive 的 Web UI 支持拖拽,支持分片上传,大文件不会断。

**场景 3:把手机当临时文件服务器。** 朋友来家里想给你传几部电影,你把手机开起来,他电脑浏览器扫码,直接传。不需要安装任何东西,不需要联网,不需要你的手机有 SIM 卡。

## 它不解决什么

这是它有意思的地方——**HcqDrive 故意不做很多事**:

- 不做云同步(那是 Syncthing 的活)
- 不做跨网络访问(那是 Tailscale / ZeroTier 的活)
- 不做账号体系(那是 Nextcloud 的活)
- 不做加密传输(那需要在自签证书和便利性之间取舍)

它只做一件事:**在同一个 WiFi 下,让手机和别的设备能用最快的方式互传文件**,并且**把这件事做到不破坏现有的工作流**。

## 怎么用

装上之后,界面上就一个"启动服务"按钮。点了之后:
- 通知栏出现一个常驻通知,显示 6 位配对码和访问地址
- 屏幕中央出现二维码
- 同一 WiFi 的任何设备浏览器访问那个地址,输入配对码就能进

进入之后,你会看到一个熟悉的文件管理器界面:左边是目录树,右边是文件列表。支持搜索、排序、右键菜单、拖拽上传、缩略图预览。基本上就是 macOS Finder 的安卓文件版。

## 和其他工具的差别

| 工具 | 它做什么 | HcqDrive 不做的事 |
|---|---|---|
| **LocalSend** | 一次性互传单文件 | 持续的文件浏览/管理 |
| **PairDrop** | AirDrop 替代(WebRTC) | 需要客户端登录 |
| **KDE Connect** | 设备互联全家桶 | 需要每台设备都装 KDE 应用 |
| **Nextcloud** | 自托管云盘 | 需要服务器、有学习成本 |
| **Syncthing** | 持续文件同步 | 双向同步,不是浏览 |

HcqDrive 的定位是:**手机就是服务器,浏览器就是客户端,开箱即用**。

## 技术

- **Android**:Kotlin 2.0 + Jetpack Compose + Ktor 3.0(CIO 引擎做 server)+ ZXing(二维码)+ Apache Commons Compress(ZIP)。无 DI 框架、无 Room、无 Java。
- **Web**:Vue 3 + Vite + TypeScript + Tailwind + Pinia,响应式 + 暗色模式。
- 19 个 REST 端点,完整契约在 `docs/api-contract.md`(1700 行)。

M1 完整功能(配对、文件管理、断点下载、分片上传、缩略图、EXIF)已经全做完,代码 MIT 开源。
